# 复兴机械厂网站

> 基于纯前端技术的企业展示网站，支持跨设备数据同步、多语言切换和管理后台

## 项目简介

复兴机械厂网站是一个现代化的企业展示平台，包含客户访问页面和管理后台两个部分。项目采用纯前端技术栈（HTML5 + CSS3 + JavaScript ES6+），通过 JSONBin.io 云端服务实现跨设备数据同步，无需自建服务器。

### 核心特性

- **跨设备数据同步** — 手机提交留言，电脑管理后台即时查看
- **云端图片管理** — 管理后台配置图片，客户页面实时更新（支持上传和链接两种方式）
- **双端自适应** — 单一 HTML 文件，自动检测设备，分别渲染桌面 Web 版和移动 H5 版
- **三语切换** — 支持中文 / English / 日本語 实时切换
- **H5 移动端优化** — 底部 Tab 导航、抽屉菜单、下滑关闭模态框
- **离线缓存保护** — 网络异常时自动降级到 localStorage，恢复后可同步至云端
- **零服务器成本** — 使用 JSONBin.io 免费服务，无需运维

## 快速开始

### 1. 本地预览

直接双击打开 HTML 文件即可在浏览器中预览：

```bash
# 客户访问页面
open index.html

# 管理后台
open admin.html
```

> **注意：** 直接打开 HTML 文件时，移动端设备检测仍可正常工作（桌面浏览器默认加载 Web 版）。

### 2. 启动本地服务器（推荐）

为了让手机等设备能够访问，建议启动本地 HTTP 服务器：

**使用 Python 3：**
```bash
cd /path/to/fuxingjixie
python3 -m http.server 8080
```

**使用 Node.js：**
```bash
npm install -g http-server
http-server -p 8080
```

**使用 VS Code：**
1. 安装 "Live Server" 扩展
2. 右键 `index.html` → "Open with Live Server"

启动后访问：
- 电脑端：`http://localhost:8080/index.html`
- 手机端：`http://你的电脑IP:8080/index.html`（自动加载 H5 版）
- 管理后台：`http://localhost:8080/admin.html`

### 3. 管理后台登录

- **访问地址：** `admin.html`
- **默认密码：** `fuxing@2024`
- **功能模块：**
  - 数据概览 — 留言统计、未处理提醒、最新5条留言
  - 留言管理 — 搜索、筛选、查看详情、标记处理、导出CSV、删除
  - 图片管理 — 上传/链接方式配置网站图片（支持14个图片位）
  - 系统设置 — 修改管理密码

## 云端配置

项目默认使用 JSONBin.io 作为云端数据存储服务。如需启用跨设备同步功能，请按以下步骤配置：

### 配置步骤

1. **注册 JSONBin.io 账号**
   - 访问 https://jsonbin.io
   - 使用 GitHub/Google 账号或邮箱注册

2. **获取 API Key**
   - 在 Dashboard 找到 "API Keys" 部分
   - 复制 X-Master-Key（格式：`$2a$10$xxxxxxxxx`）

3. **创建数据仓库（Bin）**
   - 留言数据 Bin：`{"messages": []}`
   - 图片配置 Bin：`{"images": {}}`
   - 记录两个 Bin 的 ID

4. **修改配置文件**
   
   需要在 **3 个位置** 更新 `CLOUD_CONFIG`：

   | 文件 | 位置 | 说明 |
   |------|------|------|
   | `js/cloud-service.js` | 第 4-9 行 | 两端共享的云端服务 |
   | `index.html` 内嵌脚本 | — | 页面内联配置 |
   | `admin.html` 内嵌脚本 | 第 336 行 | 管理后台配置 |

   配置格式：
   ```javascript
   const CLOUD_CONFIG = {
       API_KEY: '你的API_KEY',
       MESSAGES_BIN_ID: '留言Bin_ID',
       IMAGES_BIN_ID: '图片Bin_ID'
   };
   ```

5. **测试验证**
   - 手机提交留言 → 电脑后台查看
   - 电脑配置图片 → 手机刷新显示

详细配置步骤请参考：[doc/配置指南.md](doc/配置指南.md)

## 技术栈

| 类别 | 技术 | 说明 |
|------|------|------|
| 前端框架 | 原生 HTML5 + CSS3 + JavaScript (ES6+) | 零依赖，无需构建工具 |
| 云端存储 | JSONBin.io (REST API v3) | 留言数据和图片配置的跨设备同步 |
| 数据格式 | JSON | 云端数据存储格式 |
| 响应式方案 | 设备检测 + 双端独立布局 | 非 Media Queries，而是 UA + innerWidth 判断后加载不同 CSS/JS |
| 图片处理 | Canvas API | 自动压缩（最大宽度 800px，质量 70%，JPEG 格式） |
| 本地缓存 | localStorage | 离线降级、图片缓存、密码存储 |
| 多语言 | data-lang 属性 + JS 对象映射 | 支持中文 / English / 日本語 |
| 模态框 | 自定义 Overlay + CSS 动画 | Web 端点击遮罩关闭，H5 端支持下滑手势关闭 |
| 平滑滚动 | scrollBehavior API + requestAnimationFrame 降级 | 兼容不支持原生平滑滚动的旧浏览器 |

### 脚本加载顺序（index.html）

```
1. cloud-service.js   ← 云端服务（两端共享，最先加载）
2. web.js 或 h5.js    ← 根据设备类型动态加载其中一个
3. resize-watch.js    ← 窗口 resize 监听（两端共享）
4. admin-bridge.js    ← 图片桥接（两端共享，须在 DOM 就绪后执行）
```

## 数据流架构

```
                          JSONBin.io (云端)
                          ┌──────────────┐
                  ┌──────│  Messages Bin │──────┐
                  │      │  Images Bin   │      │
                  │      └──────┬───────┘      │
            GET /latest              PUT         GET /latest
                  │                      │              │
                  ▼                      ▼              ▼
┌──────────────┐    REST API    ┌──────────────┐    REST API    ┌──────────────┐
│  index.html  │◄─────────────►│  cloud-svc.js│◄─────────────►│  admin.html  │
│  (客户页面)   │               │  (云端服务层)  │               │  (管理后台)   │
└──────┬───────┘               └──────────────┘               └──────┬───────┘
       │                                                             │
       │  ┌─────────────┐                                           │
       ├──│ web.js       │ (桌面端)                                  │
       ├──│ h5.js        │ (移动端)                                  │
       └──│admin-bridge.js│ (图片桥接)                               │
       │                                                             │
       └──────────────► localStorage ◄──────────────────────────────┘
                    (离线缓存 / 降级保护 / 图片缓存)
```

### 数据流向说明

1. **留言提交流程：** 客户页面提交留言 → `cloud-service.js` 异步写入 Messages Bin → 管理后台刷新时读取
2. **图片配置流程：** 管理后台上传/输入图片 → 写入 Images Bin → 客户页面 `admin-bridge.js` 读取并渲染
3. **离线降级流程：** 云端不可用时 → 数据写入 localStorage → 页面加载时优先读取云端，失败则读取本地缓存
4. **双端图片同步：** `admin-bridge.js` 同时处理 Web 端（通过 ID 查找 `fx-img-{id}`）和 H5 端（通过 CSS 类名映射）的图片渲染

## 成本说明

### JSONBin.io 免费额度

| 资源 | 免费额度 | 说明 |
|------|---------|------|
| API 请求 | 100,000 次/月 | 每次留言提交/读取约 2 次请求 |
| 数据仓库 | 100 个 Bin | 本项目使用 2 个（留言 + 图片） |
| 单 Bin 容量 | 100 KB | 图片配置建议使用 URL 链接而非 Base64 上传 |
| 预估使用量 | 约 3,000 次/月 | 每天约 50 条留言 × 2 次请求 × 30 天 |

**结论：** 免费额度完全够用，无需付费升级。建议图片管理优先使用 URL 链接方式，避免 Base64 编码占用过多 Bin 容量。

## 注意事项

1. **配置一致性**
   - `js/cloud-service.js`、`index.html` 内嵌脚本、`admin.html` 内嵌脚本 三处的 `CLOUD_CONFIG` 必须完全一致
   - 修改配置时请同步更新所有位置

2. **图片大小限制**
   - 上传的图片会自动压缩（最大宽度 800px，JPEG 质量 70%）
   - 单个上传文件限制 500KB（压缩前），超过会提示压缩
   - 图片配置总数据限制 90KB（JSONBin 单 Bin 上限 100KB，留 10KB 余量）
   - 建议图片管理优先使用 **URL 链接方式**，避免 Base64 编码超出容量
   - 推荐图片宽高比 16:9，格式 JPG/PNG/WebP

3. **数据安全**
   - API Key 会暴露在前端代码中
   - 对于小型项目这是可接受的
   - 如需更高安全性，建议使用 Cloudflare Workers 代理

4. **离线使用**
   - 未配置云端服务时，数据仅保存在本地 localStorage
   - 同设备访问正常，但无法跨设备同步
   - 云端恢复后，本地缓存数据不会自动上传

5. **设备切换**
   - `resize-watch.js` 监听窗口尺寸变化，当设备类型切换时自动刷新页面
   - 桌面浏览器缩小窗口至 768px 以下不会自动切换到 H5 版

6. **浏览器兼容性**
   - 平滑滚动在旧浏览器中自动降级为 `requestAnimationFrame` 动画
   - H5 端禁止了双击缩放，避免误操作
   - 全局函数通过 `window.xxx = xxx` 暴露，兼容 Safari 块级函数作用域问题

如遇到问题，请按照以下步骤排查：

1. **查看浏览器控制台**（F12）的错误信息
2. **检查云端配置**是否正确（3 个位置的 CLOUD_CONFIG 是否一致）
3. **测试网络连接**是否正常
4. **清除 localStorage** 后重新测试
5. **参考文档**中的常见问题解答
