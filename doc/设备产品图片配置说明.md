# 设备/产品图片配置说明

## 图片系统架构

复兴机械厂的图片系统采用两层架构：

### 第一层：设备/产品数据层 (display_items 表)
- 存储设备/产品的基本信息（标题、描述、规格等）
- 通过 `image_key` 字段关联图片（如 `equip-cnc1`、`prod-1`）
- 可选：直接存储 `image_url` 字段（外部图片链接）

### 第二层：图片配置层 (images_config 表)
- 存储实际的图片 URL
- 通过 `key` 字段与 `display_items.image_key` 对应
- 管理后台的"页面图片"功能负责维护这一层

## 图片配置流程

### 方式一：通过"页面图片"功能配置（推荐）

这是最简单的方式，适合使用项目默认图片 key 的情况：

1. **打开管理后台**
   - 访问 `admin.html` 并登录

2. **进入"页面图片"标签**
   - 点击左侧导航栏的"🖼️ 页面图片"

3. **找到对应的图片槽位**
   - 设备图片：`equip-cnc1` ~ `equip-edm1` (共7个)
   - 产品图片：`prod-1` ~ `prod-8` (共8个)

4. **上传图片**
   - 点击对应槽位的"更换图片"按钮
   - 选择本地图片文件上传
   - 图片会自动保存到云端

5. **验证效果**
   - 刷新主页 `index.html`
   - 查看设备和产品区域是否显示新图片

### 方式二：直接填写图片 URL

如果需要从外部 CDN 或其他来源使用图片：

1. **打开管理后台**
   - 访问 `admin.html` 并登录

2. **进入"设备管理"或"产品管理"**
   - 点击对应的管理标签

3. **编辑设备/产品**
   - 点击"编辑"按钮

4. **填写图片 URL**
   - 在"图片URL"字段中输入完整的图片地址
   - 例如：`https://example.com/images/cnc-machine.jpg`
   - 图片预览区域会实时显示

5. **保存**
   - 点击"保存"按钮
   - 系统会优先使用 `image_url`，如果为空则使用 `image_key`

## 常见问题

### Q1: 为什么设备管理页面显示"key: xxx 需在页面图片中配置"？

**原因**: 该设备/产品记录只有 `image_key` 字段，但 `images_config` 表中没有对应的图片 URL。

**解决方案**: 
1. 切换到"页面图片"标签
2. 找到对应的 key（如 `equip-cnc1`）
3. 上传对应的图片

### Q2: 如何添加自定义设备/产品并使用自己的图片？

**步骤**:
1. 在"设备管理"或"产品管理"中点击"添加"
2. 填写基本信息
3. 在"图片Key"字段中输入自定义 key（如 `my-equip-1`）
4. **或者**直接在"图片URL"字段中输入外部图片链接
5. 保存

**如果使用 image_key**:
- 需要到"页面图片"标签添加对应的图片配置
- 注意：新增的 key 不会自动出现在"页面图片"列表中
- 需要修改 `admin.html` 中的 `IMG_SLOTS` 数组来添加新的图片槽位

**如果使用 image_url**:
- 无需额外配置，图片会直接显示

### Q3: 图片上传后主页没有更新？

**可能原因**:
1. 浏览器缓存未清除
2. 云端数据同步延迟

**解决方案**:
1. 强制刷新主页：`Ctrl + Shift + R` (Windows) 或 `Cmd + Shift + R` (Mac)
2. 清除浏览器缓存后重新访问
3. 等待几秒后再次刷新

### Q4: 如何查看当前使用的图片配置？

**方法一**: 在管理后台的"页面图片"标签查看

**方法二**: 打开浏览器控制台，输入：
```javascript
JSON.parse(localStorage.getItem('fx_images'))
```

**方法三**: 直接在 Supabase Dashboard 中查询 `images_config` 表

## 技术细节

### 数据流向

```
用户上传/配置图片
    ↓
images_config 表 (key + url)
    ↓
display_items 表 (image_key)
    ↓
admin-bridge.js 自动关联
    ↓
主页显示图片
```

### 代码实现

**admin-bridge.js** 负责将两层数据关联：
```javascript
function applyDisplayItemImages(items) {
    items.forEach(function(item) {
        if (!item.image_url || !item.image_key) return;
        
        // 通过 image_key 查找 DOM 元素
        var webEl = document.getElementById('fx-img-' + item.image_key);
        if (webEl) {
            // 应用 image_url 作为背景图
            webEl.style.backgroundImage = 'url("' + item.image_url + '")';
        }
    });
}
```

### 优先级规则

系统按以下优先级显示图片：
1. **最高**: `display_items.image_url` (直接配置的 URL)
2. **其次**: `images_config[image_key]` (通过页面图片功能配置)
3. **最低**: 默认占位符（emoji 或灰色背景）

## 最佳实践

1. **使用页面图片功能管理默认图片**
   - 保持 `image_key` 的命名规范
   - 便于统一管理和批量更新

2. **外部图片使用 image_url**
   - 适合临时性或第三方图片
   - 不需要修改 `IMG_SLOTS` 配置

3. **定期备份图片配置**
   - 在 Supabase Dashboard 中导出 `images_config` 表
   - 保存重要的图片 URL

4. **图片优化建议**
   - 尺寸：建议 800×600 或更高
   - 格式：JPG（照片）或 PNG（图形）
   - 大小：控制在 500KB 以内
   - CDN：考虑使用 CDN 加速图片加载
