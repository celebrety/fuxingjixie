## 常见问题和解决方案

### 问题 1: display_items 表不存在

**症状**:
- 控制台显示 `404` 错误
- 错误信息包含 `relation "display_items" does not exist`

**解决方案**:
1. 打开 Supabase Dashboard: https://supabase.com/dashboard/project/sjpkjfpaocswveydbxvc
2. 点击左侧菜单 "SQL Editor"
3. 打开项目中的文件: `doc/supabase-schema-upgrade.sql`
4. 复制全部内容
5. 粘贴到 SQL Editor
6. 点击 "Run" 执行
7. 等待执行完成（应该显示 "Success. No rows returned"）
8. 验证：
   - 点击左侧 "Table Editor"
   - 确认 `display_items` 表已创建
   - 确认表中有 14 条记录（6个设备 + 8个产品）

### 问题 2: RLS 策略未配置

**症状**:
- 控制台显示 `401` 或 `403` 错误
- 错误信息包含 `new row violates row-level security policy`

**解决方案**:
执行上面的 SQL 脚本会自动配置 RLS 策略。如果已执行但仍报错：
1. 在 Supabase Dashboard 中打开 `display_items` 表
2. 点击 "Policies" 标签
3. 确认有以下 4 个策略：
   - Allow public read access
   - Allow public insert access
   - Allow public update access
   - Allow public delete access

### 问题 3: 数据为空

**症状**:
- 页面显示"暂无数据"
- 控制台显示 `加载成功，数据条数: 0`

**解决方案**:
执行上面的 SQL 脚本会插入初始数据。如果已执行但数据为空：
1. 在 Supabase Dashboard 中打开 `display_items` 表
2. 查看是否有数据
3. 如果没有，重新执行 SQL 脚本中的 INSERT 语句

### 问题 4: CLOUD_CONFIG 未定义

**症状**:
- 控制台显示 `CLOUD_CONFIG is not defined`

**解决方案**:
检查 `admin.html` 文件中是否有以下代码（应该在内联 script 中）：
```javascript
const CLOUD_CONFIG = {
    SUPABASE_URL: 'https://sjpkjfpaocswveydbxvc.supabase.co',
    SUPABASE_KEY: 'sb_publishable_RbIuhqGsWbNzfNE-cXiZNA_7BR1_1fM'
};
```

### 问题 5: 浏览器缓存问题

**症状**:
- 代码已更新但问题依然存在

**解决方案**:
1. 强制刷新浏览器缓存
   - Windows/Linux: `Ctrl + Shift + R`
   - Mac: `Cmd + Shift + R`
2. 或者清除浏览器缓存后重新加载

## 数据库表结构参考

```sql
CREATE TABLE display_items (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  category VARCHAR(50) NOT NULL,        -- 'equipment' 或 'product'
  sort_order INTEGER DEFAULT 0,         -- 排序序号
  title_zh VARCHAR(200) NOT NULL,       -- 中文标题
  title_en VARCHAR(200),                -- 英文标题
  title_ja VARCHAR(200),                -- 日文标题
  description_zh TEXT,                  -- 中文描述
  description_en TEXT,                  -- 英文描述
  description_ja TEXT,                  -- 日文描述
  image_url TEXT,                       -- 图片URL
  image_key VARCHAR(100),               -- 兼容旧系统的key
  specs JSONB DEFAULT '[]'::jsonb,      -- 规格参数
  tags JSONB DEFAULT '[]'::jsonb,       -- 标签
  is_active BOOLEAN DEFAULT true,       -- 是否启用
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);
```
